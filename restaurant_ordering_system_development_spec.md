# 早餐店線上點餐系統開發說明文件

## 1. 專案概述

### 專案名稱
早餐店線上點餐系統

### 專案背景
取代傳統人工點餐流程，降低點餐錯誤率與人力負擔。

### 專案目標
顧客透過掃描桌上靜態 QR Code，免登入即可自助點餐；店家透過後台即時接單與管理菜單，提升整體出餐效率。

### 技術定位
純前端應用搭配 Firebase 服務，無需自建後端伺服器。

---

## 2. 系統目標使用者

### 顧客端
- 掃描桌上靜態 QR Code 進入點餐頁（無須登入）
- 系統自動從網址讀取並顯示桌號
- 依分類瀏覽商品、加入購物車、填寫備註、送出訂單

### 後台使用者（店員 / 管理員，共用同一組帳號登入）
- 即時接收新訂單、新訂單視覺提示
- 查看訂單內容、切換訂單狀態
- 商品管理、分類管理
- 營業額與訂單報表

---

## 3. 系統功能需求

### 3.1 顧客點餐功能

**功能列表**
- 掃描桌上靜態 QR Code 進入點餐頁
- 系統自動從網址讀取並顯示桌號
- 依分類瀏覽商品（每項商品顯示圖片、名稱、價格、簡短描述）
- 商品加入購物車、修改數量、刪除商品
- 填寫備註（例：不要辣）
- 訂單確認頁
- 送出訂單

**流程**
1. 顧客掃描 QR Code（網址如：`/order?table=A01`）
2. 系統從網址參數讀取桌號
3. 顯示菜單（依分類）
4. 顧客選購商品並填寫備註
5. 確認訂單後送出
6. 顯示訂單送出成功畫面

---

### 3.2 後台功能（需登入）

**訂單管理**
- 即時訂單列表（Firestore 即時監聽）
- 新訂單視覺提示
- 查看訂單明細
- 訂單狀態切換：待處理 → 製作中 → 已完成 → 已取餐
- 刪除異常訂單

**商品管理**
- 新增 / 修改 / 刪除商品
- 商品欄位：名稱、價格、分類、圖片、描述
- 商品上下架
- 圖片上傳至 Firebase Storage

**分類管理**
- 新增 / 修改 / 刪除分類
- 調整分類排序

**報表**
- 今日營業額、今日訂單數
- 熱銷商品排行
- 月報表

## 6. 資料庫設計

### categories（分類）

```json
{
  "id": "c001",
  "name": "早餐",
  "sortOrder": 1
}
```

### products（商品）

```json
{
  "id": "p001",
  "name": "培根蛋餅",
  "description": "香脆培根加蛋",
  "price": 45,
  "categoryId": "c001",
  "imageUrl": "https://...",
  "isAvailable": true,
  "sortOrder": 1
}
```

### orders（訂單）

```json
{
  "id": "o001",
  "tableNo": "A01",
  "status": "pending | preparing | ready | picked_up",
  "total": 120,
  "createdAt": "timestamp"
}
```

### orders/{orderId}/items（訂單細項，子集合）

```json
{
  "id": "i001",
  "productId": "p001",
  "name": "培根蛋餅",
  "price": 45,
  "qty": 2,
  "note": "不要辣"
}
```

> `name` 與 `price` 為下單當下的快照，確保商品改價後不影響舊訂單記錄。

## 7. UI 頁面規劃

### 顧客端

| 頁面 | 路由 | 說明 |
|------|------|------|
| 菜單頁 | `/order?table=A01` | 依分類顯示商品，每項顯示圖片、名稱、價格、簡短描述 |
| 購物車 | `/order/cart` | 調整數量、填寫備註、確認訂單 |
| 送出成功 | `/order/success` | 訂單送出成功提示 |

### 後台（需登入）

| 頁面 | 路由 | 說明 |
|------|------|------|
| 登入 | `/login` | Email + 密碼登入 |
| 訂單管理 | `/admin/orders` | 即時訂單列表與狀態切換 |
| 商品管理 | `/admin/products` | 新增、修改、刪除、上下架 |
| 分類管理 | `/admin/categories` | 新增、修改、刪除、排序 |
| 報表 | `/admin/reports` | 營業額、訂單數、熱銷排行 |

## 8. 權限設計

### 角色與權限

| 角色 | 登入狀態 | 可存取功能 |
|------|----------|------------|
| 顧客 | 未登入 | 讀取菜單與分類、送出訂單 |
| 後台使用者 | 已登入 | 全部功能 |

### Firestore Security Rules

| 集合 | 顧客（未登入） | 後台使用者（已登入） |
|------|---------------|-------------------|
| `categories` | 讀取 | 讀取、寫入 |
| `products` | 讀取 | 讀取、寫入 |
| `orders` | 新增 | 讀取、寫入、刪除 |
| `orders/items` | 新增 | 讀取、寫入、刪除 |

## 9. 開發時程

### 前置作業（2～3 天）
- 安裝 Firebase CLI
- 建立 Firebase 專案
- 建立 Vue 3 專案（Vite + Tailwind + daisyUI）
- 串接 Firebase 至專案
- 建立 Firestore 初始資料（分類、測試商品）

### 第一階段（1 週）：顧客點餐
- 從網址讀取桌號
- 依分類顯示菜單
- 商品加入購物車
- 購物車頁（調整數量、填寫備註）
- 送出訂單至 Firestore

### 第二階段（1 週）：後台 - 訂單管理
- 登入頁（Firebase Auth）
- 即時訂單列表（Firestore 監聽）
- 新訂單視覺提示
- 訂單狀態切換

### 第三階段（1 週）：後台 - 商品與分類管理
- 商品新增、修改、刪除、上下架
- 商品圖片上傳（Firebase Storage）
- 分類新增、修改、刪除、排序

### 第四階段（1 週）：報表、測試與部署
- 報表頁（營業額、訂單數、熱銷排行）
- QR Code 產生（每桌列印用）
- 手機瀏覽器測試
- Firestore Security Rules 設定
- Firebase Hosting 部署

**總工期：約 4 週 + 前置 2～3 天**
